# Project Asset Hierarchy

```
projects/
  web-dev/
    <project-slug>/image-1.svg
  frontend/
    <project-slug>/image-1.svg
  ai-ml/
    (future projects)
```

- Use lowercase, hyphenated folder names for each project slug.
- Drop screenshots or hero illustrations as `image-1.(png|svg)`, `image-2.(png|svg)`, etc.
- Update the project case study data if you change filenames.
